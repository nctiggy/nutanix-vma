# nutanix-vma - Agent Guide

## Project Overview

Kubernetes operator to migrate Nutanix AHV VMs to KubeVirt. Uses kubebuilder v4 with controller-runtime.

## Build Commands

```bash
make build              # Build manager binary (runs generate + manifests + fmt + vet first)
make test               # Run unit tests with envtest
make test-integration   # Run integration tests only
make lint               # Run golangci-lint
make lint-fix           # Run golangci-lint with auto-fix
make manifests          # Regenerate CRDs and RBAC from kubebuilder markers
make generate           # Regenerate DeepCopy methods
make docker-build       # Build Docker image
make docker-push        # Push Docker image
```

## Key Design Decisions

1. **Single API group**: `vma.nutanix.io/v1alpha1` for all CRDs
2. **Custom Nutanix client**: `net/http` based, NOT the auto-generated Go SDK
3. **CDI for disk transfer**: DataVolumes with HTTP source pointing to Prism image download
4. **No virt-v2v**: AHV uses KVM/QEMU natively, same as KubeVirt -- no driver conversion needed
5. **Idempotent reconciliation**: Each phase checks for existing resources before creating; resource UUIDs persisted in status
6. **Mock server for testing**: In-process httptest server with configurable fixtures

## Testing Conventions

- **Unit tests**: `*_test.go` files alongside source, use `httptest` for HTTP mocking
- **Integration tests**: `test/integration/`, use envtest (real K8s API + etcd) + mock Nutanix server
- **E2E tests**: `test/e2e/`, require real Nutanix cluster (gated behind `NUTANIX_E2E=true`)
- **Test framework**: Ginkgo v2 + Gomega for integration/E2E, standard `testing` for unit tests

## Critical Rules

### Never Edit Auto-Generated Files
- `config/crd/bases/*.yaml` -- regenerated by `make manifests`
- `config/rbac/role.yaml` -- regenerated by `make manifests`
- `**/zz_generated.*.go` -- regenerated by `make generate`

### Never Remove Scaffold Markers
`// +kubebuilder:scaffold:*` comments are used by kubebuilder CLI.

### After Editing `*_types.go`
Always run: `make manifests && make generate`

### After Editing Any `.go` File
Run: `make build && make test`
